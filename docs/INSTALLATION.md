# Mavi VPN - Installation Guide

This guide describes in detail how to set up Mavi VPN. The system primarily consists of two components:
1. **VPN Server** (running via Docker on a Linux server)
2. **VPN Client** (e.g., for Windows)

---

## 1. Server Installation (Docker)

The VPN server is designed to be deployed on a remote server and is easiest to run using Docker.

### Prerequisites
* A server with **Docker** and **Docker Compose** installed.
* An open UDP port in your server's firewall (Default: `10433`).

### Step-by-Step Guide

1. **Prepare the Source Code:**
   Place the Mavi VPN source code on your server (e.g., via Git clone).

2. **Navigate to the backend directory:**
   ```bash
   cd backend
   ```

3. **Configure Environment Variables:**
   Create the configuration file from the template:
   ```bash
   cp .env.example .env
   ```
   Open the `.env` file and adjust at least these critical values:
   * `VPN_AUTH_TOKEN`: Set a **long and secure password** here! Every client that wants to connect will need this token.
   * `VPN_PORT`: The external UDP port your server will be reachable on (Default: `10433`).
   * `VPN_CENSORSHIP_RESISTANT`: Set this to `true` or `false` – depending on whether you want to use Layer-7 obfuscation. This value must match on both the client and the server!

4. **Start the Service:**
   Start the server in the background (detached mode). Docker will build the server code based on the provided Rust image:
   ```bash
   docker-compose up -d --build
   ```
   *(Note: The server will generate any necessary certificates on startup and store them persistently in the `data` subdirectory.)*

5. **Retrieve the Certificate PIN:**
   For a secure connection, the client needs the certificate PIN. This is generated by the server and placed in the `data` folder on first startup:
   ```bash
   cat data/cert_pin.txt
   ```
   *(Copy this value for your client configuration!)*

## 2. Windows Client Installation

The Windows client is an interactive command-line application built on the modern QUIC protocol, using WinTUN for the network interface.

### Prerequisites

1. **Rust (Cargo):** Required on your Windows system to compile the client from source code.
2. **Wintun Driver:**
   The program requires `wintun.dll` to create the tunnel interface.
   * You can find it in the `windows/` folder. If it's missing, download it directly from [wintun.net](https://www.wintun.net/).
   * The `wintun.dll` file **must** be placed in the exact same folder as the final `.exe` executable.

### Step-by-Step Guide

1. **Compile the Windows Client:**
   Open a PowerShell console on your development machine in the project root directory (where the main `Cargo.toml` file is located) and compile the Windows application:
   ```powershell
   cargo build --release -p windows-vpn
   ```
   *(This will create the `.exe` file in the `target/release/` folder after some time.)*

2. **Assemble the Files:**
   It is recommended to create a dedicated installation folder for the VPN on your file system (e.g., `C:\MaviVPN\`).
   Copy the following files there:
   * The newly compiled `target\release\mavi-vpn.exe`
   * The driver file `wintun.dll`

3. **MANDATORY: Run as Administrator!**
   The VPN client deeply integrates with the Windows networking system to modify routes and provide the virtual tunnel interface (TUN). **If started without administrator privileges, it will fail immediately.**
   * Search for PowerShell or Command Prompt in your Windows Start menu.
   * **Right-click** it and explicitly select **"Run as administrator"**.
   * Navigate to your new VPN folder in the terminal:
     ```powershell
     cd C:\MaviVPN\
     ```
   * Start the application:
     ```powershell
     .\mavi-vpn.exe
     ```

4. **First Connection and Setup:**
   When you start the client for the first time, it will guide you interactively through the setup and save the details in a `config.json` file, so you don't have to enter them on every start. It looks something like this:
   ```text
   ╔══════════════════════════════════════╗
   ║         Mavi VPN - Windows           ║
   ╚══════════════════════════════════════╝

   Server Endpoint (e.g., vpn.example.com:443): your-server-ip:10433
   Auth Token: [The exact same token as in your backend .env]
   Certificate PIN (SHA256 hex): [The PIN from data/cert_pin.txt on the server]
   Censorship Resistant Mode? [y/N]: [Answer matching your server's .env]
   ```

5. **VPN Disconnect:**
   To safely disconnect the VPN and restore your Windows routing to its previous normal state, return to the console window where the VPN is running and press `Ctrl + C`.

### Troubleshooting / Debugging
If the connection fails or drops unexpectedly, you can start the client in a verbose mode for deeper log insights. Open an admin console again and run:
```powershell
$env:RUST_LOG = "debug"
.\mavi-vpn.exe
```
This will provide detailed information on whether the QUIC handshake succeeds or if there are permission issues when creating the WinTUN adapter.
